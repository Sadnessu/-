我构建了一个基于边缘AI的"语音+视觉"多模态智能家居中控Agent系统。它解决了传统智能家居依赖固定语音指令和App操作、无法理解用户模糊意图的核心痛点，实现了"一句话+一个手势"即可触发复杂场景联动的自然交互体验，且全部推理在本地完成，彻底消除隐私泄露风险。
核心痛点： 现有方案存在三个明显缺陷。第一，语音助手只能识别预设指令，用户说"家里有点闷"时无法理解其对应"开窗+开空调+开净化器"的复合需求，需要用户逐项手动操作；第二，纯语音交互在嘈杂环境或隐私场景下体验差，缺乏视觉补充，且误唤醒率高；第三，主流方案依赖云端处理，存在网络延迟和隐私泄露风险，尤其涉及室内摄像头数据时用户顾虑较大，断网时直接瘫痪。
系统架构与核心逻辑流： 整个系统采用"端侧多模态感知→意图理解Agent→设备控制Agent→执行反馈闭环"的长链推理架构，三个Agent通过共享内存池协作，形成完整的感知-决策-执行链路。
在感知层，我基于RK3588国产芯片设计了本地中控硬件，集成6麦克风环形阵列实现5米远场拾音和180度声源定位，同时搭载1080P摄像头用于手势识别和场景理解。语音和视觉数据在本地进行特征提取，通过硬件NPU加速，无需上传云端。针对RK3588的NPU特性，我将视觉模型从FP32量化至INT8，语音模型采用动态INT8量化，在精度损失小于2%的前提下，推理速度提升3.8倍，内存占用降低60%。
核心是多模态意图理解Agent。它采用"编码器-融合器-解码器"结构：语音分支使用Whisper Tiny进行ASR转写和情感分析，识别用户情绪状态（如疲惫、兴奋）；视觉分支使用轻量版YOLOv8-nano进行15类手势检测和人体姿态估计，包括"挥手""指向""遮挡眼睛"等语义手势。两个模态的特征在交叉注意力融合层进行对齐，通过门控机制动态调整模态权重。当用户说"太亮了"并同时做出"遮挡眼睛"的手势时，Agent能融合两个模态的信息，以94%的置信度推断出"关闭主灯+调暗氛围灯+拉上窗帘"的复合意图，而非仅执行单一的调光操作。该Agent基于Qwen-1.8B进行LoRA指令微调，结合知识蒸馏技术压缩至1.2B参数，在保持端侧实时推理（平均响应<500ms）的同时，支持模糊意图的理解和澄清对话。对于超出当前能力的请求，Agent会主动发起多轮对话澄清，而非直接执行错误指令。
设备控制Agent负责将高层意图拆解为可执行动作序列。它维护着家中所有设备的实时状态图谱（照明、空调、窗帘、新风、电视等），通过规则引擎+近端策略优化（PPO）算法进行动作规划。规则引擎处理确定性逻辑（如安全约束：燃气报警时禁止开灯），PPO网络优化用户体验。例如接收到"我要看电影"的意图后，Agent会自动执行：关闭主灯→调暗氛围灯至20%暖光→关闭窗帘→开启电视并切换到观影模式→空调调至静音模式→新风系统切换至内循环。整个过程通过Home Assistant协议与各类设备通信，支持Zigbee 3.0、WiFi 6、蓝牙Mesh多协议接入，并具备设备离线自动重连和状态同步机制。
系统还包含一个持续学习Agent，它会记录用户的修正反馈（如用户手动调亮了灯光或撤销了某个动作），通过在线强化学习更新PPO策略网络，使场景联动越来越贴合个人习惯。所有学习过程均在本地完成，原始数据不出设备，仅上传加密后的梯度摘要用于家庭间联邦学习（可选关闭）。
多Agent协作机制： 三个Agent通过基于Redis的共享内存池进行协作，延迟低于5ms。感知Agent将结构化特征写入内存，意图Agent读取并输出意图标签和设备参数，控制Agent执行后返回状态码，形成完整闭环。当某个Agent置信度不足时（如手势被遮挡导致视觉置信度<0.6），会触发协作协商机制，由意图Agent主动发起语音澄清："请问您是指关闭客厅灯还是所有灯光？"若3秒内无响应，则执行保守策略（仅关闭当前房间灯光）。
落地成果与性能数据： 目前该系统已在个人家庭环境完成部署，覆盖客厅、卧室和书房三个区域，接入设备18台。端到端响应延迟平均380ms（感知120ms+意图260ms），意图识别准确率达到91%，手势识别准确率96%，多模态融合后复合意图理解准确率提升至94%。整机功耗控制在8W以内，其中NPU推理功耗2.3W。相比传统语音助手，复杂场景的配置效率提升约70%，用户日均交互次数从3.2次提升至11.5次。每日消耗约200万Token用于本地推理，等效云端API调用成本节省约85%。后续计划扩展至厨房场景，加入烟雾/溢锅视觉检测，并支持多用户声纹识别实现个性化策略隔离。
